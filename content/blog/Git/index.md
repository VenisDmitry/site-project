## Что такое система управления версиями?

Система управления версиями (VCS) — это инструмент, который помогает отслеживать изменения в файлах, сохранять историю и работать в команде.

Она позволяет:
- возвращаться к прошлым версиям
- сравнивать изменения
- работать нескольким разработчикам одновременно

## Виды систем контроля версий

- Локальные (Local VCS)
- Централизованные (CVCS)
- Распределённые (DVCS)

> [!NOTE]
> Git относится к распределённым системам контроля версий (DVCS).

## Что такое Git?

Git — это распределённая система управления версиями, созданная Линусом Торвальдсом.

Основные особенности:
- высокая скорость работы
- работа без постоянного подключения к серверу
- гибкая система ветвления

## Основные команды Git


git init        # создать репозиторий
git clone       # клонировать репозиторий
git add         # добавить файл в индекс
git commit      # сохранить изменения
git status      # проверить статус
git push        # отправить изменения
git pull        # получить изменения


## Жизненный цикл файлов в Git

- Untracked (неотслеживаемые)
- Modified (изменённые)
- Staged (подготовленные)
- Committed (сохранённые)

## Ветвление (Branching)

Git позволяет создавать ветки для разработки новых функций:


git branch feature
git checkout feature


Или сразу:


git checkout -b feature


Слияние веток:


git merge feature


## Mindmap


- Git
  - Основы
    - init
    - clone
    - status
  - Работа с файлами
    - add
    - commit
  - Работа с сервером
    - push
    - pull
  - Ветки
    - branch
    - merge


## Полезные команды


git log          # история коммитов
git diff         # различия
git reset        # откат изменений
git stash        # временно сохранить изменения


## Преимущества Git

- Бесплатный и open-source
- Поддержка командной работы
- Гибкость и масштабируемость

> [!WARNING]
> Неправильное использование команд вроде git reset --hard может привести к потере данных!

## Заключение

Git — один из самых важных инструментов для разработчика. Он используется практически во всех проектах и является базовым навыком для DevOps, backend и frontend разработчиков.
